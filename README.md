# CarouselBeats

Instagram carousel to playlist web app built with Django.

## Deploy on Render

1. Push this repo to GitHub.
2. In Render, choose `New +` -> `Blueprint`.
3. Select the repository so Render picks up [`render.yaml`](/c:/Users/think/OneDrive/Desktop/Projects/insta-carousel-playlist/render.yaml).
4. Deploy the web service.

## Environment Variables

Render will create most of these from [`render.yaml`](/c:/Users/think/OneDrive/Desktop/Projects/insta-carousel-playlist/render.yaml):

- `DJANGO_SECRET_KEY`: autogenerated on first deploy.
- `DEBUG=False`
- `ALLOWED_HOSTS=.onrender.com`
- `MEDIA_ROOT=/tmp/media`

Optional:

- `INSTAGRAM_USERNAME`
- `INSTAGRAM_PASSWORD`
- `CSRF_TRUSTED_ORIGINS`

## Notes

- The app runs as a single Docker web service.
- Background processing uses in-process threads, so it is best suited to one running instance.
- Generated media is stored in `/tmp/media`, which is ephemeral on Render. Files disappear after restarts or redeploys.
- `ffmpeg` is installed through the Docker image, so no extra buildpacks are needed.

## Local Run

```bash
pip install -r requirements.txt
python manage.py runserver
```
